# 继承 vs 装饰器

## 继承

### 使用方式

- 通过重写父类的某些方法

### 优点

- 大部分语言内置

### 缺点

- 可能错误的覆盖父类的某些方法或属性(特别是对于Python等动态语言)
- 依赖父类的实现，直接调用特定方法的时候并不能生效


## 装饰器(Python)


### 使用方式

- 在原来的方法前面加装饰器，会生成一个新的方法覆盖原来的方法，见附件

### 优点

- 对原来的方法改动小
- 直观，很像代理，但是非常轻量级

### 缺点

- 少数语言支持(Python),其实就是一个简单的语法糖，可以使用类或者闭包模拟

## 附件

### Python的装饰器

```

def hello(fn):
    def newfunc():
        print("hello")
        return fn()
    # 返回新的函数
    return newfunc
    
@hello
def empty():
    pass
    
# 等价于 empty = hello(empty)
empty() # >>> hello

```
